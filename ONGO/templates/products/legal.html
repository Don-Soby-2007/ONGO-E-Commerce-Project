This page explains the terms of using the ONGO demo e-commerce platform and how user information is handled.

1. About This Project

ONGO is a demo e-commerce project built strictly for learning and portfolio purposes. It is not a real commercial store.

  • This website is a demonstration of web development skills and platform architecture.
  • No real payments are processed on this site.
  • Payment integrations (Razorpay) are used only in test mode for demonstration of the checkout flow.

Developed and maintained by:

Don Soby

GitHub LinkedIn

2. Terms of Use

By accessing this platform, users are permitted to utilize the following features for testing purposes:

  • Browse and search through our mock product catalog.
  • Add items to a persistent shopping cart and manage wishlists.
  • Place demonstration orders to experience the workflow.
  • Manage personal profile information and address books.

Note: Orders placed on this site do not represent real purchases. No items will be shipped, and no real currency will be exchanged. The platform is intended for demonstration and educational use only.

3. Account Information

Users may create and access accounts through:

  • Standard Email/Password authentication.
  • Google OAuth 2.0 authentication.

When using Google login, the following information may be retrieved from your Google account:

  • Username / Full Name
  • Profile Picture
  • Email Address
  • Phone Number (if provided and shared by Google)

This data is used only to create and manage the user account inside the platform and to provide a personalized user experience within this demo environment.

4. Privacy Policy

As a demo platform, we store basic user data to maintain session continuity and simulate a real e-commerce experience. Data stored includes:

Data Collected

  • • Email & Username
  • • Profile Photo
  • • Shipping & Billing Addresses
  • • Order History & Preferences

Usage Purpose

  • • Account Management
  • • Order Processing Simulation
  • • UX Personalization

We do not sell, rent, or share your data with third parties for marketing purposes.

5. Payment Disclaimer

This website uses Razorpay test mode for demonstration of the payment flow.

  • No real payments are ever processed.
  • Any payment pages shown are mock testing interfaces only.
  • Do not enter actual credit card, debit card, or net banking credentials. Use only the provided test credentials in the Razorpay sandbox.

6. Data Security

User data is handled securely within the context of this application. We implement industry-standard authentication and access controls to protect your dummy account data.

Warning: Since this is a portfolio/demo project, it should not be used to store sensitive, critical, or real-world private data.

7. Cookies & Tracking

The platform uses basic session cookies for:

  • Maintaining login sessions.
  • Cart and session management.
  • Security and authentication.

We do not use third-party advertising or tracking cookies.
